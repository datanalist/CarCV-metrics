## 1. Введение

### 1.1 Назначение документа

Данный документ описывает системный дизайн бортовой системы компьютерного зрения CARS (Computer Automotive Recognition System) для распознавания транспортных средств в режиме реального времени. Документ предназначен для инженеров, архитекторов и заинтересованных сторон проекта.

### 1.2 Область применения

Система предназначена для:
- Контроля доступа на объекты — автоматическое распознавание автомобилей сотрудников/гостей
- Мониторинга парковки — учёт въезда/выезда транспорта
- Обеспечения безопасности — поиск угнанных/разыскиваемых ТС
- Аналитики трафика — статистика по типам и маркам автомобилей
- Логистики — отслеживание грузового транспорта

### 1.3 Целевая аудитория

- Охранные предприятия
- Парковочные комплексы
- Логистические компании
- Государственные структуры (ДПС, охрана объектов)

## 2. Цели и бизнес-контекст

### 2.1 Бизнес-цели

| Цель | Описание | Метрика успеха |
|------|----------|----------------|
| **Автономное распознавание ТС** | Детекция и идентификация транспортных средств в режиме реального времени на борту автомобиля | Точность детекции >90%, задержка <50ms |
| **Распознавание номерных знаков** | OCR номерных знаков с поддержкой российского алфавита | Точность распознавания >85% |
| **Классификация ТС** | Определение марки, типа кузова и цвета автомобиля | Точность классификации >75% |
| **Автономная работа** | Функционирование без подключения к облаку/серверу | Uptime >99%, работа при отсутствии связи |
| **Энергоэффективность** | Работа от бортовой сети автомобиля (12V) | Потребление <25W |

### 2.2 Обоснование проекта

**Преимущества бортовой системы:**

- Автономность (не требует постоянного подключения к сети)
- Мобильность (может работать на патрульных автомобилях, эвакуаторах)
- Низкая стоимость владения (нет затрат на облачные сервисы)
- Конфиденциальность данных (обработка локально)

### 2.3 Ключевые стейкхолдеры

| Роль | Интересы | Влияние |
|------|----------|---------|
| Оператор системы | Простота использования, надёжность | Высокое |
| IT-администратор | Обслуживание, интеграция | Среднее |
| Служба безопасности | Точность распознавания, отчётность | Высокое |
| Руководство | ROI, масштабируемость | Высокое |

## 3. Требования к системе

### 3.1 Функциональные требования

| ID | Требование | Приоритет | Описание |
|----|------------|-----------|----------|

### 4.5 Структура базы данных

**Схема таблицы `data`:**

```sql
CREATE TABLE data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp INTEGER NOT NULL,       -- Unix timestamp (секунды)
    track_id INTEGER NOT NULL,        -- ID объекта от трекера
    cx REAL,                          -- Центр X bbox (0.0-1.0)
    cy REAL,                          -- Центр Y bbox (0.0-1.0)
    w REAL,                           -- Ширина bbox (0.0-1.0)
    h REAL,                           -- Высота bbox (0.0-1.0)
    car_color TEXT,                   -- Распознанный цвет (15 классов)
    car_model TEXT,                   -- Тип кузова (6 классов)
    car_make TEXT,                    -- Марка автомобиля (20 классов)
    face_count INTEGER DEFAULT 0,     -- Количество обнаруженных лиц
    face_coords TEXT,                 -- Координаты лиц (CSV)
    lp_number TEXT,                   -- Распознанный номер
    status INTEGER DEFAULT 0          -- Статус обработки (0=new, 1=processed)
);

-- Индексы для оптимизации запросов
CREATE INDEX idx_timestamp ON data(timestamp);
CREATE INDEX idx_track_id ON data(track_id);
CREATE INDEX idx_status ON data(status);
CREATE INDEX idx_lp_number ON data(lp_number);
```

### 5.3 Расчёт объёма хранилища

**Сценарий: Патрульный автомобиль (8 часов работы/день)**

```
Средняя интенсивность: 10 автомобилей/минуту
За 8 часов: 10 × 60 × 8 = 4,800 детекций

Хранилище на детекцию:
- БД запись: ~1 KB
- Изображение авто: ~432 KB
- Изображение номера: ~27 KB
- Изображение лица (50%): ~50 KB × 0.5 = 25 KB
- ИТОГО: ~485 KB

За день: 4,800 × 485 KB ≈ 2.3 GB
За неделю: 2.3 × 7 ≈ 16 GB
За месяц: 2.3 × 30 ≈ 70 GB
```

**Рекомендация:** NVMe SSD 128-256 GB с автоматической ротацией данных

## 11. Риски и ограничения

### 11.1 Технические риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| **Thermal throttling** | Средняя | Высокое | Активное охлаждение, мониторинг температуры |
| **OOM (Out of Memory)** | Низкая | Высокое | Swap файл, оптимизация batch size |
| **Отказ камеры** | Средняя | Высокое | Резервная камера, auto-reconnect |
| **Потеря питания** | Средняя | Среднее | UPS, graceful shutdown |
| **Переполнение хранилища** | Высокая | Среднее | Автоматическая ротация данных |
| **Сбой RTSP** | Средняя | Среднее | Автоматический reconnect |

### 11.2 Операционные риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| **Вибрации в автомобиле** | Высокая | Среднее | Амортизирующее крепление |
| **Экстремальные температуры** | Средняя | Высокое | Термозащитный корпус |
| **Отсутствие сети** | Высокая | Низкое | Автономная работа, локальное хранение |
| **Повреждение при ДТП** | Низкая | Высокое | Безопасное размещение, backup данных |

### 11.3 Известные ограничения

#### Функциональные ограничения

1. **Одновременные потоки:** только 1 камера (расширение требует рефакторинга)
2. **Алфавит номеров:** только русский (0-9, A-Z RU subset)
3. **Ночная съёмка:** требуется IR подсветка
4. **Скорость движения:** OCR эффективен до 60 км/ч

#### Аппаратные ограничения

1. **RAM:** 8 GB (ограничивает количество моделей)
2. **Хранилище:** зависит от SSD (рекомендуется 128+ GB)
3. **Питание:** максимум 25W (ограничение Jetson)

### 11.4 Зависимости от внешних систем

| Зависимость | Критичность | Fallback |
|-------------|-------------|----------|
| NVIDIA TensorRT | Критическая | Нет (обязательно) |
| DeepStream SDK | Критическая | Нет (обязательно) |
| ONNX Runtime | Высокая | CPU fallback |
| SQLite | Высокая | Файловое хранение |
| Камера | Критическая | Резервная камера |
