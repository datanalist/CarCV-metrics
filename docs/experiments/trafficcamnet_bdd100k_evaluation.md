# Оценка TrafficCamNet на BDD100K (val, car)

**Дата:** 26 января 2026  
**Статус:** ✅ Эксперимент завершен  
**Автор:** CarCV Team

---

## Обзор

Проведена оценка качества детектора TrafficCamNet на датасете BDD100K (split `val`) по классу `car`. Оценка выполнена без DeepStream, с использованием ONNX Runtime + CUDA и собственного декодирования выходов модели. Рассчитаны метрики качества (mAP, precision/recall/F1, mean IoU) и производительности (inference time, FPS).

## Методология

1. Загружены аннотации BDD100K `val` и отфильтрованы объекты класса `car`.
2. Выполнен инференс TrafficCamNet с декодированием выходов и NMS.
3. Для каждого изображения сохранены предсказанные боксы в координатах пикселей и время инференса.
4. Метрики качества посчитаны по IoU-порогам `0.5..0.95` с шагом `0.05`.

Ключевые реализации:
- Инференс и декодирование: `scripts/trafficcamnet_bdd100k/inference.py`
- Оценка метрик и сбор статистики: `scripts/trafficcamnet_bdd100k/evaluate.py`

## Наблюдения

- Высокий precision при низком recall указывает на заметное количество пропусков объектов.
- Mean IoU для корректных совпадений остается высоким, что говорит о приемлемой локализации при успешных детекциях.
