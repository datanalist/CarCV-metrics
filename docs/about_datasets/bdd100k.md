# BDD100K Dataset

## Общая информация

## Формат изображений

### Именование файлов
Формат: `{hash}-{hash}.jpg`

Примеры:
- `0000f77c-6257be58.jpg`
- `b1c66a42-6f7d68ca.jpg`

### Разделение данных
- **Train:** ~70,000 изображений
- **Val:** ~10,000 изображений  
- **Test:** ~20,000 изображений (без публичных аннотаций)

## Формат аннотаций (детекция объектов)

### Структура JSON файла

Каждый JSON файл содержит массив объектов, где каждый объект описывает одно изображение:

```json
{
    "name": "0000f77c-6257be58.jpg",
    "attributes": {
        "weather": "clear",
        "scene": "city street",
        "timeofday": "daytime"
    },
    "timestamp": 10000,
    "labels": [
        {
            "category": "car",
            "attributes": {
                "occluded": false,
                "truncated": false,
                "trafficLightColor": "none"
            },
            "manualShape": true,
            "manualAttributes": true,
            "box2d": {
                "x1": 45.24,
                "y1": 254.53,
                "x2": 357.80,
                "y2": 487.90
            },
            "id": 0
        }
    ]
}
```

### Метаданные изображения

#### Атрибут `weather`
- `clear` — ясная погода
- `overcast` — облачно
- `rainy` — дождь
- `snowy` — снег
- `foggy` — туман
- `undefined` — не определено

#### Атрибут `scene`
- `highway` — шоссе
- `residential` — жилой район
- `city street` — городская улица
- `parking lot` — парковка
- `gas stations` — заправочные станции
- `tunnel` — туннель
- `undefined` — не определено

#### Атрибут `timeofday`
- `dawn/dusk` — рассвет/закат
- `daytime` — дневное время
- `night` — ночь

### Категории объектов

#### Транспортные средства
- `car` — легковой автомобиль
- `truck` — грузовик
- `bus` — автобус
- `bike` — велосипед
- `motor` — мотоцикл

#### Люди
- `person` — пешеход
- `rider` — водитель/пассажир

#### Инфраструктура
- `traffic light` — светофор
- `traffic sign` — дорожный знак

### Формат bounding box

Координаты в формате `box2d`:
- `x1` — левая координата (пиксели)
- `y1` — верхняя координата (пиксели)
- `x2` — правая координата (пиксели)
- `y2` — нижняя координата (пиксели)

**Важно:** Координаты абсолютные, в пикселях изображения.

### Атрибуты объектов

- `occluded` (bool) — объект частично закрыт другими объектами
- `truncated` (bool) — объект обрезан краем кадра
- `trafficLightColor` (str) — цвет светофора:
  - `none` — не применимо
  - `red` — красный
  - `yellow` — жёлтый
  - `green` — зелёный

## Использование для CarCV

### Применимость

BDD100K подходит для:
- ✅ Обучение/валидация детектора автомобилей (TrafficCamNet)
- ✅ Оценка качества детекции в разных условиях (погода, время суток, тип сцены)
- ✅ Анализ распределения классов и метаданных
- ✅ Бенчмаркинг производительности моделей

### Ограничения

- ❌ Нет аннотаций номерных знаков
- ❌ Нет классификации марок/моделей автомобилей
- ❌ Нет аннотаций лиц водителей
- ❌ Тестовый набор без публичных аннотаций

### Рекомендации по использованию

1. **Парсинг аннотаций:**
   - Загрузить JSON файлы
   - Сопоставить изображения с аннотациями по имени файла (`name`)

2. **Конвертация координат:**
   - BDD100K использует формат `{x1, y1, x2, y2}` (абсолютные пиксели)
   - Для YOLO: конвертировать в `{cx, cy, w, h}` (нормализованные 0-1)
   - Для COCO: конвертировать в `{x, y, w, h}` (абсолютные пиксели)

3. **Фильтрация по метаданным:**
   - Использовать `attributes` для анализа производительности в разных условиях
   - Фильтровать по `weather`, `scene`, `timeofday` для целевых экспериментов

4. **Валидация:**
   - Использовать `val` набор для оценки качества моделей
   - Учитывать метаданные при интерпретации результатов

```python
import json

with open('/home/mk/Загрузки/DATASETS/bdd100k/bdd100k_labels_release/bdd100k/labels/bdd100k_labels_images_train.json') as f:
    annotations = json.load(f)

car_images = [
    ann for ann in annotations 
    if any(label['category'] == 'car' for label in ann['labels'])
]
```

### Конвертация координат в YOLO формат

```python
def bdd100k_to_yolo(box2d, img_width, img_height):
    """Конвертирует box2d в YOLO формат (нормализованные cx, cy, w, h)"""
    x1, y1, x2, y2 = box2d['x1'], box2d['y1'], box2d['x2'], box2d['y2']
    
    cx = (x1 + x2) / 2 / img_width
    cy = (y1 + y2) / 2 / img_height
    w = (x2 - x1) / img_width
    h = (y2 - y1) / img_height
    
    return cx, cy, w, h
```
