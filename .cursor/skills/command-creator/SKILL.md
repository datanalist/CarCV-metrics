# Command Creator

Create reusable Cursor commands — Markdown-файлы в `.cursor/commands/`, доступные через `/` в чате.

## Что такое команды

Команды — это Markdown-файлы с инструкциями для AI-агента. При вводе `/` в чате Cursor показывает список доступных команд. Выбранная команда вставляется как промпт.

## Расположение

| Тип | Путь | Область |
|---|---|---|
| Проектные | `.cursor/commands/` | Репозиторий, расшаривать через git |
| Глобальные | `~/.cursor/commands/` | Все проекты на машине |

## Процесс создания

### 1. Определить цель

Выяснить у пользователя:
- Что должна делать команда?
- Когда и как часто будет использоваться?
- Какой уровень детализации нужен?

### 2. Выбрать паттерн

| Паттерн | Когда использовать | Пример |
|---|---|---|
| **Пошаговый** | Сложные многоэтапные процессы | code-review, debug-issue |
| **Директивный** | Простые одноразовые задачи | deslop, clarify-task |
| **Шаблонный** | Стандартизированный вывод | git-commit, create-pr |
| **Визуальный** | Генерация диаграмм/визуализаций | diagrams, overview |

### 3. Написать команду

Структура файла зависит от паттерна. Общие правила:

- **Имя файла** — kebab-case, описательное: `code-review.md`, `run-all-tests-and-fix.md`
- **Заголовок H1** — название команды, совпадает с назначением
- **Императивный стиль** — "Выполни", "Проверь", "Создай" (не "Выполняется", "Проверяется")
- **Конкретные инструкции** — включать команды CLI, форматы, ограничения
- **Чеклист** — для сложных команд добавить `- [ ]` список для самопроверки

### 4. Проверить

1. Создать файл в `.cursor/commands/`
2. Набрать `/` в чате Cursor — команда должна появиться
3. Выполнить команду, проверить результат
4. Доработать по результатам

## Структура по паттернам

### Пошаговый (Step-by-step)

```markdown
# Название команды

## Overview

Краткое описание (1-2 предложения).

## Steps

1. **Этап 1**
   - Действие
   - Действие
2. **Этап 2**
   - Действие
   - Действие

## Checklist

- [ ] Критерий 1
- [ ] Критерий 2
```

### Директивный (Directive)

```markdown
# Название команды

Прямая инструкция без секций.

Это включает:
- Пункт 1
- Пункт 2

В конце сообщить результат в 1-3 предложения.
```

### Шаблонный (Template)

```markdown
# Название команды

## Overview

Краткое описание.

## Steps

1. **Подготовка** — команды для сбора данных
2. **Генерация** — создание результата

## Template

`формат шаблона`

## Rules

- Правило 1
- Правило 2
```

### Визуальный (Visual)

```markdown
# Название команды

## Overview

Что визуализировать.

## Instructions

1. Анализ входных данных
2. Выбор типа диаграммы
3. Генерация

## Output

Формат вывода (Mermaid, таблица и т.д.)
```

## Рекомендации

- **Длина**: 20-80 строк — достаточно деталей без перегрузки контекста
- **Одна задача**: каждая команда решает одну конкретную задачу
- **CLI-команды**: включать точные команды (`git diff --cached`, `ruff check`)
- **Без чатовости**: команды — это инструкции, не диалоги
- **Примеры**: добавлять конкретные примеры вывода, если формат важен
