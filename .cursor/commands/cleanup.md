# Очистка проекта от неиспользуемых файлов

## Overview

Найди и удали файлы, которые не задействованы в плане docs/rules/plan.md, нигде не упоминаются и ни на что не ссылаются в проекте. Цель — убрать мёртвый код, забытые артефакты и осиротевшие файлы.

## Steps

1. **Сканирование проекта**
   - Собери полный список файлов проекта (исключая `.git/`, `.venv/`, `node_modules/`, `__pycache__/`, `.cursor/`)
   - Раздели файлы по категориям: код (`*.py`), данные (`data/`), модели (`models/`), документация (`docs/`), конфиги, скрипты, ноутбуки

2. **Анализ связей**
   - Проверь файлы, которые не используются для выполнения плана
   - Для каждого файла проверь, упоминается ли его имя или путь в других файлах проекта через поиск по содержимому
   - Проверь импорты Python (`import`, `from ... import`)
   - Проверь ссылки в Markdown (`@путь`, `[text](путь)`, бэктики с путями)
   - Проверь ссылки в конфигах (`*.yaml`, `*.json`, `*.toml`, `*.cfg`)
   - Проверь использование в shell-скриптах и Makefile

3. **Классификация результатов**
   - Для каждого потенциально неиспользуемого файла определи уровень уверенности:
     - **Точно не используется** — имя файла нигде не встречается в проекте
     - **Вероятно не используется** — упоминается только в `.gitignore` или комментариях
     - **Требует проверки** — упоминается косвенно или неочевидным образом

4. **Отчёт пользователю**
   - Выведи таблицу найденных файлов с колонками: путь, размер, категория, уровень уверенности
   - Для каждого файла кратко объясни, почему считаешь его неиспользуемым
   - Спроси пользователя, какие файлы удалить

5. **Удаление**
   - Удали только те файлы, которые пользователь подтвердил
   - Выведи итог: сколько файлов удалено, сколько места освобождено

## Rules

- НЕ удаляй файлы без явного подтверждения пользователя
- НЕ трогай: `.git/`, `.venv/`, `.env`, `.gitignore`, `pyproject.toml`, `uv.lock`, `README.md`
- НЕ считай неиспользуемыми: `__init__.py`, конфиги в корне проекта, `.cursor/`
- Файлы в `data/` проверяй особенно внимательно — они могут загружаться динамически
- Пустые директории после удаления файлов тоже предложи удалить
