# Актуализация правил проекта

## Overview

Проверь все `.cursor/rules/*.mdc` на соответствие текущему состоянию проекта и предложи обновления. Цель — правила должны отражать реальную структуру, стек, процессы и соглашения.

## Steps

1. **Сбор текущего состояния**
   - Прочитай все файлы в `.cursor/rules/`
   - Прочитай все файлы в `.cursor/agents/`
   - Прочитай все `SKILL.md` из `.cursor/skills/*/`
   - Прочитай `docs/architecture.md`, `docs/rules/task.md`, `docs/rules/plan.md`, `README.md`
   - Просканируй структуру проекта: `ls -R` верхнего уровня и ключевых директорий (`scripts/`, `models/`, `data/`, `docs/`, `notebooks/`)
   - Проверь `pyproject.toml` на актуальные зависимости и настройки

2. **Анализ расхождений**
   - Для каждого rule-файла проверь:
     - **Стек**: упомянутые библиотеки/версии совпадают с `pyproject.toml`?
     - **Структура**: упомянутые папки и файлы существуют?
     - **Процессы**: описанные workflow соответствуют реальным скриптам и конфигам?
     - **Ссылки**: все `@path` и `[text](path)` ведут на существующие файлы?
     - **Метрики/пороги**: актуальны ли целевые значения?
     - **Инструменты**: CLI-команды работают? Нет устаревших?
     - **Дублирование**: содержит ли rule инструкции, workflow или контент, который уже полностью описан в `.cursor/agents/` или `.cursor/skills/`? Если да — кандидат на удаление из rule.

3. **Отчёт**
   - Выведи таблицу расхождений: файл правила → что устарело / дублируется → предлагаемое изменение → покрыто агентом/скиллом
   - Отметь критичность: 🔴 ошибка (битая ссылка, несуществующий путь), 🟡 устарело (версия, метрика), 🟢 косметика, 🔵 дубликат (контент уже покрыт агентом или скиллом)
   - Если расхождений нет — сообщи что правила актуальны

4. **Применение**
   - Спроси пользователя, какие изменения применить
   - Внеси только подтверждённые правки
   - При дублировании (🔵): удали секцию из rule; если нужен ориентир — замени короткой ссылкой на агент/скилл
   - Выведи diff каждого изменённого файла

## Rules

- НЕ меняй правила без подтверждения пользователя
- НЕ добавляй новые правила — только актуализируй существующие
- НЕ удаляй секции без подтверждения; исключение — подтверждённые дубликаты (🔵), покрытые агентом или скиллом
- Сохраняй формат и стиль каждого rule-файла
- Если нашёл проблему, которую нельзя решить правкой правил (например, отсутствует целый документ) — сообщи отдельно
