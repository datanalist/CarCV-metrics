# Актуализация документации проекта

## Overview

Проверь всю документацию проекта на актуальность и приведи в соответствие с текущим состоянием кодовой базы. Обнови устаревшие ссылки, метрики, статусы и структуру.

## Steps

1. **Сканирование кодовой базы**
   - Собери текущее состояние: модели в `models/`, скрипты в `scripts/`, ноутбуки в `notebooks/`, результаты в `results/`, конфиги в `configs/`
   - Найди все Python-файлы с функциональным кодом и проверь наличие docstrings
   - Проверь наличие новых/удалённых файлов, которые не отражены в документации

2. **Аудит `docs/architecture.md`**
   - Сверь список моделей и их метрики с реальными файлами в `models/`
   - Проверь актуальность pipeline flow, API endpoints, конфигураций DeepStream
   - Обнови секцию "Обновления и история изменений" — добавь новые записи
   - Обнови секцию "Следующие шаги" — отметь завершённые, добавь новые
   - Проверь ссылки на ноутбуки, отчёты и результаты — убедись что файлы существуют
   - Обнови версию документа и дату

3. **Аудит `README.md`**
   - Сверь секцию "Модели ML" с актуальными метриками из экспериментов
   - Обнови "Текущее состояние" — завершённые/текущие/запланированные задачи
   - Проверь секцию "Документация" — все ссылки на docs/ должны быть рабочими
   - Обнови Roadmap в соответствии с реальным прогрессом
   - Обнови "Ключевые достижения" если есть новые результаты
   - Обнови версию и дату

4. **Аудит `docs/rules/plan.md`**
   - Проверь, что все завершённые шаги отмечены `[x]`
   - Убедись, что текущий шаг соответствует `docs/rules/task.md`
   - Удали устаревшие пункты если задачи отменены

5. **Аудит экспериментов и датасетов**
   - Проверь `docs/experiments/` — для каждого завершённого эксперимента должен быть отчёт
   - Проверь `docs/about_datasets/` — для каждого используемого датасета должна быть документация
   - Проверь `docs/datasets_and_models_overview.md` — актуальность списков
   - Проверь `docs/system-design/ML_System_Design_Document.md` — соответствие требованиям

6. **Проверка битых ссылок**
   - Во всех `.md` файлах проверь внутренние ссылки: `[text](path)`, `@path`, `` `path` ``
   - Убедись, что все указанные пути существуют на диске
   - Исправь или удали битые ссылки

7. **Отчёт**
   - Выведи таблицу изменений: файл, что обновлено, причина
   - Если ничего не изменилось — сообщи что документация актуальна

## Checklist

- [ ] `docs/architecture.md` — метрики, модели, pipeline актуальны
- [ ] `README.md` — статусы, метрики, roadmap актуальны
- [ ] `docs/rules/plan.md` — шаги отмечены корректно
- [ ] `docs/experiments/` — отчёты существуют для завершённых экспериментов
- [ ] `docs/about_datasets/` — документация есть для используемых датасетов
- [ ] Все внутренние ссылки в `.md` файлах рабочие
- [ ] Версии и даты обновления актуальны во всех документах
- [ ] Нет дублирующейся или противоречивой информации между документами
