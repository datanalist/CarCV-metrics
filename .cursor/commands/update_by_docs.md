# Обновление документа по контексту из других файлов

## Usage

```
/cleanup doc_1 doc_2 ... doc_n target_doc
```

Аргументы передаются через пробел. Последний аргумент — `target_doc` (файл для обновления). Все предыдущие — источники контекста.

**Пример:**
```
/cleanup docs/experiments/eval.md scripts/inference.py docs/rules/plan.md
```

Обновит `docs/rules/plan.md` на основе контекста из `docs/experiments/eval.md` и `scripts/inference.py`.

---

## Instructions

Пользователь передал аргументы: `{{input}}`

Выполни следующее:

1. **Разбор аргументов**
   - Разбей `{{input}}` по пробелам
   - Последний элемент — это `target_doc` (файл, который нужно обновить)
   - Все остальные элементы — это `source_docs` (источники контекста)

2. **Чтение файлов**
   - Прочитай `target_doc` целиком — это документ, который нужно привести в соответствие
   - Прочитай каждый файл из `source_docs` — это источники актуальной информации
   - НЕ анализируй остальную файловую структуру проекта

3. **Анализ расхождений**
   - Сравни содержимое `target_doc` с информацией из `source_docs`
   - Найди устаревшие, неточные или отсутствующие данные в `target_doc`:
     - Упоминания путей, имён функций, классов, переменных — они изменились?
     - Описания поведения — они соответствуют коду в `source_docs`?
     - Метрики, параметры, конфигурации — они актуальны?
     - Шаги, инструкции, примеры — они всё ещё корректны?

4. **Обновление**
   - Внеси изменения в `target_doc`, сохраняя его структуру и стиль
   - Обновляй только то, что явно противоречит или устарело относительно `source_docs`
   - Не добавляй информацию, которой нет в `source_docs`

5. **Отчёт**
   - Перечисли все внесённые изменения в формате "было → стало"
   - Если расхождений не найдено — сообщи об этом явно

## Rules

- Читай ТОЛЬКО файлы из аргументов, не сканируй проект
- Сохраняй язык и стиль оригинального `target_doc`
- Не переписывай документ с нуля — только точечные правки
- Если аргументы не переданы — выведи Usage и завершись
