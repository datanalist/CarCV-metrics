# Git: Reset to Commit

## Overview

Перевести текущую ветку (working dir, index, HEAD) в состояние указанного коммита.

## Steps

### 1. Узнать хеш коммита

- `git log --oneline -20` — последние коммиты
- `git log --oneline -n 1 <branch>` — хеш HEAD ветки
- Пользователь может передать хеш или ref (например `HEAD~2`)

### 2. Выбрать режим reset

| Режим | Команда | Working dir | Index | HEAD |
|-------|---------|-------------|-------|------|
| **--hard** | Полное совпадение с коммитом | ✓ перезаписывается | ✓ перезаписывается | откат |
| **--mixed** (default) | Изменения остаются в working dir | изменения сохранены | сброшен | откат |
| **--soft** | Изменения staged | изменения staged | откат | откат |

**По умолчанию** предлагать `--hard`, если пользователь явно не просит сохранить изменения.

### 3. Выполнить reset

```bash
git reset --hard <commit>
```

или

```bash
git reset --mixed <commit>
git reset --soft <commit>
```

### 4. Подтверждение

- Для `--hard` — **обязательно** спросить подтверждение: изменения будут безвозвратно потеряны
- Перечислить, какие коммиты исчезнут из текущей ветки: `git log <commit>..HEAD`

## Rules

- Не выполнять без явного указания пользователем хеша/ref коммита
- Для `--hard` — всегда запрашивать подтверждение
- Если коммит уже запушен — предупредить: для push потребуется `--force-with-lease`
- Не запускать force push без разрешения пользователя
